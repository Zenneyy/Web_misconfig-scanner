# 🛡️ Web Misconfig Scanner

A modern, production-grade vulnerability scanner designed to detect common web security misconfigurations. Built with a robust **Node.js** backend and a professional **React (Vite + TypeScript)** dashboard, this tool helps developers and security professionals audit web applications for compliance with OWASP standards.

![Security](https://img.shields.io/badge/OWASP-A02%3A2025-blue)
![License](https://img.shields.io/badge/License-MIT-green)

---

## 🚀 Key Features

* **Deep Crawling Engine:** Recursive scanning capabilities with Sitemap parsing and `robots.txt` awareness.
* **Comprehensive Security Checks:**
    * **Headers:** HSTS, CSP, X-Frame-Options, X-Content-Type-Options, Referrer-Policy.
    * **Cookies:** Detection of missing Secure/HttpOnly flags and unsafe SameSite configurations.
    * **Sensitive Files:** Probes for exposed `.env`, `.git`, backups (`.zip`, `.sql`), and config files.
    * **CORS:** Analysis of wildcard origins and credential exposure risks.
    * **Information Disclosure:** Detection of stack traces, debug errors, and server version headers.
* **Safety First:** Built-in SSRF protection (blocks private/local IPs), rate limiting, and mandatory ownership confirmation.
* **Professional Reporting:**
    * **Interactive Dashboard:** Real-time scan monitoring and historical data.
    * **Detailed Findings:** Evidence-based reporting with specific remediation steps.
    * **Exportable Reports:** Download scan results as standalone HTML or JSON files.

## 🛠️ Tech Stack

### Backend
* **Runtime:** Node.js (v18+)
* **Framework:** Express.js
* **Core Logic:** Axios (HTTP), Cheerio (Parsing), Ipaddr.js (SSRF Guard)
* **Storage:** Atomic JSON-based file storage (No database installation required)

### Frontend (`/frontend`)
* **Framework:** React (Vite)
* **Language:** TypeScript
* **Styling:** Tailwind CSS
* **Components:** Radix UI / Shadcn-like components
* **Icons:** Lucide React

## ⚡ Quick Start Guide

Follow these steps to get the scanner running locally.

### 1. Clone the Repository
```bash)
git clone https://github.com/Zenneyy/Web_misconfig-scanner.git
```

### 2. Setup & Run Backend

```bash
# Navigate to backend directory
cd backend

# Install dependencies
npm install

# Start the server (Runs on http://localhost:3000)
npm run start

```

> **Note:** Keep this terminal open. You should see: `🛡️ WEB MISCONFIG SCANNER API STARTED`

### 3. Setup & Run Frontend

The frontend provides the user interface. Open a **new terminal window** for this step.

```bash
# Navigate to frontend directory
cd frontend

# Install dependencies
npm install

# Start the development server
npm run dev

```

> **Note:** The terminal will show a local URL (usually `http://localhost:8080`). Open this link in your browser.

## 📖 Usage Guide

### Running a New Scan

1. Navigate to the **Dashboard** (e.g., `http://localhost:8080`).
2. Click the **"New Scan"** button.

### Configuration

* **Target URL:** Enter the full URL (e.g., `https://example.com`).
* **Scan Depth:**
* **Level 0:** Single Page (Fast check of just the target URL).
* **Level 1:** Standard (Target + Sitemap + Direct Links).
* **Level 2:** Deep Crawl (Follows links found on Level 1 pages).


* **Authorization:** You must check the "Authorization Confirmation" box. This tool is for authorized testing only.
* Click **"Start Security Scan"**.

### Viewing Reports

* **Dashboard:** View a summary of recent scans, risk levels, and scan status.
* **Scan Details:** Click on any scan to see the full list of findings.
* **Findings Table:** Sortable list of vulnerabilities (Critical, High, Medium, Low, Info).
* **Evidence:** Expand any finding to see the exact HTTP header, cookie, or code snippet that triggered the alert.
* **Remediation:** Follow the specific advice to fix the issue.
